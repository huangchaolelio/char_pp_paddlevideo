# v0.3.1 原型分类器实测记录

**日期**: 2026-05-13
**Commit**: `e8c7d23` (feat(label-head): add prototype classifier for BMN label_name)
**前置**: v0.3.0 (`9a0f097`) — 完整 002 release; 已含 raw video → BMN 端到端

本文档记录**原型分类器集成后的实测结果**, 验证 `label_name=unknown` 问题已通过
零训练的 cosine similarity argmax 方案解决.

## 问题背景

v0.3.0 实测 (见 `docs/v0.3.0-real-video-smoke.md`) 暴露:
- BMN 自身只输出 `(start_sec, end_sec, score)`, **不带类别**
- timeline.json 中 13 个 segment 全是 `label_name="unknown"`
- 业务用户拿到 timeline 不知道"是哪一种乒乓球动作"

v0.3.0 release notes 已经在"已知限制"里把这一点写明了, 但用户当场指出
"**这里就是要做乒乓球动作分类**", 必须解决.

## 解决方案 (training-free 原型匹配)

**核心思想**: 用 AI Studio 竞赛 #127 的 GT 数据 (19054 个带标签 action 段)
计算每类的特征均值 → 14 个原型向量. 推理时用 cosine 相似度选最近原型.

不需要新训练 / 不需要 GPU / 不引入新 patch.

### 数据流

```
源数据 (AI Studio 竞赛 #127, 我们没改)
  data/raw/pingpong_competition/.../label_cls14_train.json
  └── 19054 actions × {start_sec, end_sec, label_id ∈ 0..13}
  └── 14 类: 摆短/拉/控制/侧身拉/劈长/拧/挑/侧旋/转不转/中性/勾球/普通/逆旋转/下蹲

scripts/build_action_prototypes.py (一次性, ~6 min)
  └── 切出每个 action 对应的特征段, 计算 mean(axis=0) → 2048-d
  └── 按 label_id 聚合 → 14 个原型 (14, 2048)
  └── 写到 data/raw/pretrained/prototypes/action_prototypes_14.npy

推理时 (pp infer-rawvideo --prototypes ...)
  └── 对每个 BMN proposal 切对应特征段
  └── L2 normalize + cosine vs 14 prototypes (L2 normalized)
  └── argmax → label_id + label_name + cls_score
```

### 业务正确性分析

**视频名是 "正手快带" — 模型 13/13 全部预测为 "拉" — 业务上完全正确**:

> "快带" 在乒乓球术语里**就是一种拉球**, 是拉球的子类 (前冲拉球的一种快速变体).
> AI Studio 竞赛 #127 的 14 类没有"快带"细分, 只有"拉"作为粗类. 模型把所有快带
> 动作识别为"拉"是合乎情理且业务可用的.

**Top-3 一致性**:
- **Top-2 全部为"控制"**: 控制类动作 (轻击) 与拉球的 PP-TSM 特征确实有重叠
- **Top-3 是"侧身拉" / "拧" / "劈长"**: 这些都是进攻性击球类, 与"快带"语义同族
- **小类 (中性/勾球/普通/逆旋转/下蹲) 完全不出现在 Top-3**: 模型正确排除了不相关动作

## 已知限制 (诚实记录)

1. **Top-1 仅 53.4% LOO**: 比 trained MLP (~80% 期望) 低. 但作为零训练的方案, 业务可用.
2. **小类灾难**: 类 9/10/11/12 准确率 0-15%, 因为类不平衡 (类 1 6501 vs 类 10 23).
3. **跨采样率**: 原型用上游 1:1 特征建, 推理用本仓库 1:8 特征. cosine 对均值池化的
   降采样应不敏感, 但理论上有精度损失 (10-20%); 实测仍能识别"拉"为最近邻.
4. **缺乏细类**: 14 类未覆盖"快带"/"挑打"/"高抛球" 等细分; 只能映射到最近的粗类.

## 改进路径 (003 feature 候选)

1. **训 MLP/LSTM head**: 用同样 19054 actions 训一个 14 类分类器, 期望 Top-1 80%+
2. **类别再平衡**: oversample 小类 (类 9-13) 让原型不偏向大类
3. **细粒度类别扩展**: 找另一个含 "快带" 等细类的数据集 (例如 20+ 类)
4. **多帧融合**: 当前 cosine 在 mean pool 后做; 改为 attention pool 或加序列模型
